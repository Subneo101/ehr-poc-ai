# local and remote setups and previews


## yaml file for automating deployment

You should add the YAML file right now, before you run your Git commands to create the repository.
Because we want GitHub to start building your website the very first time you upload your code, the file needs to be sitting in your project folder before you push.
Here is the exact order to do it:

## Step 1: Create the file if you haven't already
Make sure you have created the folder and the file locally on your Mac:

* Folder path: ehr-poc-ai/.github/workflows/
* File name: deploy.yml

### via CLI

 
## for creating .github folder, Use cli:

 ``` mkdir -p .github/workflows && touch .github/workflows/deploy.yml ```


## Yaml file
Contents of the YAML file is in the Deploy to GitHub Pages script.
if not below is a sample:

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches:
      - main # Triggers the build every time you push code to the main branch

# Sets permissions of the GITHUB_TOKEN to allow deployment to GitHub Pages
permissions:
  contents: read
  pages: write
  id-token: write

# Allow only one concurrent deployment to prevent overlap collisions
concurrency:
  group: "pages"
  cancel-in-progress: true

jobs:
  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Set up Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'npm'

      - name: Install dependencies
        run: npm ci # Performs a clean, automated install matching your package.json exactly

      - name: Build application
        run: npm run build

      - name: Upload production artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: './dist' # Tells GitHub to upload the folder generated by Vite

      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

## Explanation of the code

```
git init
git add .
git commit -m "chore: initial commit with deployment workflow"
gh repo create ehr-poc-ai --private --source=. --remote=origin --push
```

This single command tells the GitHub CLI (gh) to handle the entire lifecycle of creating a remote repository and linking it to your local machine all in one shot.
Instead of forcing you to go to the GitHub website, click buttons, and copy-paste URLs back into your terminal, this handles everything behind the scenes.
Here is the exact breakdown of what each piece of that command means:
## 1. gh repo create ehr-poc-ai

* What it does: This tells the GitHub CLI to connect to your online account and create a brand-new, empty code repository named ehr-poc.

## 2. --public or --private

* What it does: This sets the visibility of your online repository to Public or Private.
* Why it matters: Only you (and anyone you explicitly invite as a collaborator) will be able to see your source code, your files, or your project history on the GitHub website.

## 3. --source=.

* What it does: The dot (.) means "the current folder my terminal is sitting in."
* Why it matters: This tells the CLI tool, "Hey, don't create a blank project from scratch. Take all the code, files, and folders currently sitting in this local directory on my Mac and use them as the foundation for the repository."

## 4. --remote=origin

* What it does: This creates a Git connection link named origin.
* Why it matters: In Git, origin is the standard nickname given to the main master server on the internet where your code lives. This explicitly links your local Mac project folder to your new online ehr-poc web link, so your computer knows exactly where to send files in the future.

## 5. --push

* What it does: This immediately triggers a file upload.
* Why it matters: As soon as the repository is created and linked, this flag instantly executes a git push. It copies all your locally committed files (including your new .github/workflows/deploy.yml automation script) and uploads them directly to GitHub's servers right away.

------------------------------
## What to expect when you hit Enter:
When you run this command, your terminal will print out a few lines showing it initialization, mapping the remote target, and a progress bar as it uploads your objects. Once it finishes, it will print out a direct https://github.com web link.
